### 1.3 技术选型理由

| 技术决策 | 理由 |
|----------|------|
| Flask 而非 Django | 课程要求；轻量级，适合中小型项目 |
| SQLite 而非 MySQL | 课程要求；零配置，单文件数据库，无需安装数据库服务 |
| Flask-SQLAlchemy 而非原生 SQL | ORM 防 SQL 注入、代码可读性强、自动建表 |
| Werkzeug 哈希而非明文 | 课程要求密码安全存储；自动加盐 |
| Waitress 而非 Flask dev server | Flask 开发服务器有双导入Bug（`__main__` vs `app`），Waitress 更稳定 |
| Bootstrap CDN 而非本地 | 减少项目体积、自动更新、加载快 |
| iframe 嵌入而非自建播放器 | 课程要求"嵌入第三方播放器或跳转链接"；零带宽成本、版权合规 |
| AJAX 收藏而非表单提交 | 用户体验更好，无需刷新页面 |

### 1.4 动画与视觉效果技术

| 技术 | 用途 |
|------|------|
| CSS `@keyframes` | 浮动音符、频谱跳动、声波纹、唱片旋转、金牌闪烁、粒子爆炸 |
| CSS `backdrop-filter: blur()` | 导航栏和卡片的玻璃态毛玻璃效果 |
| CSS `transition` | 卡片悬浮、按钮交互、边框发光过渡 |
| CSS `will-change: transform` | GPU 加速提示，只对旋转唱片和封面缩放使用 |
| CSS `@media (prefers-reduced-motion)` | 无障碍：用户系统开启减少动画时自动停用 |
| JS `DocumentFragment` | 批量创建 DOM 元素，减少浏览器回流次数 |
| JS `IntersectionObserver` | 已在 JS 中预置，P2 可用于滚动懒加载 |

### 2.3 蓝图架构

项目使用 Flask **Blueprint（蓝图）** 实现模块化路由管理：

```python
# app.py 中注册
app.register_blueprint(auth_bp)      # 无前缀，/login, /register
app.register_blueprint(main_bp)      # 无前缀，/, /browse, /search, /ranking
app.register_blueprint(song_bp)      # 前缀 /song
app.register_blueprint(admin_bp)     # 前缀 /admin
app.register_blueprint(playlist_bp)  # 前缀 /playlist
app.register_blueprint(comment_bp)   # 前缀 /comment
```

蓝图的优势：
- 各模块独立，互不干扰
- 支持 URL 前缀统一管理
- 方便 P2 扩展新模块
- 可将不同蓝图部署到不同服务器

## 七、数据库设计详解

### 7.1 User 表（users）

```sql
CREATE TABLE users (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    username      VARCHAR(80)  NOT NULL UNIQUE,   -- 用户名，支持中文
    email         VARCHAR(120) NOT NULL UNIQUE,   -- 邮箱
    password_hash VARCHAR(256) NOT NULL,           -- Werkzeug 哈希密码
    avatar        VARCHAR(256) DEFAULT '',         -- 头像文件名
    bio           TEXT DEFAULT '',                 -- 个人简介
    role          VARCHAR(20)  DEFAULT 'user',     -- user / admin
    created_at    DATETIME DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX ix_users_username ON users(username);
```

### 7.2 Song 表（songs）

```sql
CREATE TABLE songs (
    id             INTEGER PRIMARY KEY AUTOINCREMENT,
    title          VARCHAR(200) NOT NULL,          -- 歌曲名称
    artist         VARCHAR(200) NOT NULL,          -- 歌手名
    album          VARCHAR(200) DEFAULT '',        -- 专辑名
    cover_image    VARCHAR(500) DEFAULT '',        -- 封面图文件名
    audio_url      VARCHAR(500) DEFAULT '',        -- 嵌入链接
    style          VARCHAR(50)  NOT NULL,          -- 风格分类
    description    TEXT DEFAULT '',                -- 歌曲简介
    uploader_id    INTEGER NOT NULL,               -- FK → users.id
    play_count     INTEGER DEFAULT 0,              -- 播放次数
    favorite_count INTEGER DEFAULT 0,              -- 收藏数（冗余，便于排序）
    comment_count  INTEGER DEFAULT 0,              -- 评论数（冗余，便于排序）
    created_at     DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (uploader_id) REFERENCES users(id) ON DELETE CASCADE
);
CREATE INDEX ix_songs_title ON songs(title);
CREATE INDEX ix_songs_artist ON songs(artist);
```

> 设计说明：`favorite_count` 和 `comment_count` 是**冗余字段**。
> 好处：排行榜排序时不需要 JOIN 统计，查询速度快。
> 维护方式：收藏/取消收藏时 `+=1`/`-=1`；发表/删除评论时 `+=1`/`-=1`。

### 7.3 Favorite 表（favorites）

```sql
CREATE TABLE favorites (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id    INTEGER NOT NULL,                   -- FK → users.id
    song_id    INTEGER NOT NULL,                   -- FK → songs.id
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (song_id) REFERENCES songs(id) ON DELETE CASCADE,
    UNIQUE (user_id, song_id)                      -- 同一用户不能重复收藏
);
```

### 7.4 Playlist 表（playlists）

```sql
CREATE TABLE playlists (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    name        VARCHAR(200) NOT NULL,             -- 歌单名称
    description TEXT DEFAULT '',                   -- 描述
    user_id     INTEGER NOT NULL,                  -- FK → users.id
    is_public   BOOLEAN DEFAULT 1,                 -- 是否公开
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
);
```

### 7.5 PlaylistSong 表（playlist_songs）

```sql
CREATE TABLE playlist_songs (
    id          INTEGER PRIMARY KEY AUTOINCREMENT,
    playlist_id INTEGER NOT NULL,                  -- FK → playlists.id
    song_id     INTEGER NOT NULL,                  -- FK → songs.id
    added_at    DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (playlist_id) REFERENCES playlists(id) ON DELETE CASCADE,
    FOREIGN KEY (song_id) REFERENCES songs(id) ON DELETE CASCADE,
    UNIQUE (playlist_id, song_id)                   -- 同一歌单不能重复添加
);
```

### 7.6 Comment 表（comments）

```sql
CREATE TABLE comments (
    id         INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id    INTEGER NOT NULL,                   -- FK → users.id
    song_id    INTEGER NOT NULL,                   -- FK → songs.id
    content    TEXT NOT NULL,                      -- 评论内容
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (song_id) REFERENCES songs(id) ON DELETE CASCADE
);
```

### 7.7 BrowseRecord 表（browse_records）

```sql
CREATE TABLE browse_records (
    id               INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id          INTEGER NOT NULL,             -- FK → users.id
    song_id          INTEGER NOT NULL,             -- FK → songs.id
    browse_count     INTEGER DEFAULT 1,            -- 浏览次数
    last_browsed_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (song_id) REFERENCES songs(id) ON DELETE CASCADE,
    UNIQUE (user_id, song_id)                       -- 同一用户-歌曲只记录一条
);
```

### 7.8 表关系 ER 图（文字版）

```
User (1) ────< (多) Song          # 一个用户上传多首歌曲
User (1) ────< (多) Favorite      # 一个用户收藏多首歌曲
User (1) ────< (多) Playlist      # 一个用户创建多个歌单
User (1) ────< (多) Comment       # 一个用户发表多条评论
User (1) ────< (多) BrowseRecord  # 一个用户浏览多首歌曲

Song (1) ────< (多) Favorite      # 一首歌被多人收藏
Song (1) ────< (多) Comment       # 一首歌有多条评论
Song (1) ────< (多) BrowseRecord  # 一首歌被多人浏览

Playlist (1) >──< (多) Song       # 多对多（通过 PlaylistSong 中间表）
```

### 9.2 权限装饰器（utils/decorators.py）

```python
from functools import wraps
from flask import abort
from flask_login import current_user

def admin_required(f):
    """管理员权限装饰器 — 非管理员访问返回 403"""
    @wraps(f)
    def decorated_function(*args, **kwargs):
        if not current_user.is_authenticated or not current_user.is_admin():
            abort(403)
        return f(*args, **kwargs)
    return decorated_function
```

### 9.3 封面上传处理（app.py 工具函数）

```python
import uuid
from werkzeug.utils import secure_filename

ALLOWED_COVER_EXTENSIONS = {'png', 'jpg', 'jpeg', 'gif', 'webp'}

def save_cover(file_storage, upload_folder):
    """保存封面图，UUID 重命名防冲突"""
    if file_storage and file_storage.filename:
        ext = file_storage.filename.rsplit('.', 1)[1].lower()
        if ext in ALLOWED_COVER_EXTENSIONS:
            filename = f"{uuid.uuid4().hex}.{ext}"
            filepath = os.path.join(upload_folder, filename)
            file_storage.save(filepath)
            return filename
    return None
```
